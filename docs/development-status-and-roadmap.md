## ネクストステップ（優先度順）

### Phase 1: プロジェクト初期化と基盤構築（1-2週間）
1. **Expoプロジェクトの初期化**
   ```bash
   npx create-expo-app sf6match-mobile --template
   ```
   
2. **基本的なプロジェクト構造の作成**
   ```
   src/
   ├── presentation/
   │   ├── screens/
   │   ├── components/
   │   └── navigation/
   ├── domain/
   │   ├── entities/
   │   ├── usecases/
   │   └── repositories/
   └── data/
       ├── repositories/
       └── datasources/
   ```

3. **必要なライブラリのインストール**

   - Redux Toolkit
   - Axios
   - AsyncStorage

## リスクと対策

1. **SF6公式サイトAPI**
   - リスク：公式APIが存在しない/制限がある
   - 対策：スクレイピングの検討、段階的なデータ取得

2. **技術的課題**
   - リスク：Convex/Mastraの学習曲線
   - 対策：プロトタイプ作成、技術検証の早期実施

3. **パフォーマンス**
   - リスク：大量データでのパフォーマンス低下
   - 対策：ページネーション、キャッシュ戦略の最適化
